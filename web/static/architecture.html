iPhone → Mac File Transfer

A terminal-based tool to browse and copy files from iPhone to Mac or any external HDD — no iTunes, no Finder, no GUI required.

Why this exists — the problem

🔒

macOS hides iPhone files

Finder shows a single "Trust" dialog and nothing more. Browsing, filtering, or batch-selecting individual files is not possible.

🐢

AirDrop & iCloud are slow

Wireless transfer of hundreds of RAW photos or long 4K videos over AirDrop or iCloud takes orders of magnitude longer than direct USB.

💿

Direct HDD copy is impossible

There is no native way to copy the camera roll directly to an external hard drive in one step — no app, no Finder feature.

Two ways to use it

⌨️

Interactive TUI

python main.py

  • Two-panel browser — iPhone on left, destination on right
  • Checkbox selection for individual files and whole folders
  • SPACE to toggle, A to select all in current directory
  • Live progress bar — per-file and overall
  • Tab to switch panels, T to start, Esc to cancel
🌐

Web UI

http://127.0.0.1:8765

  • Dark-themed browser interface — works from any device on the LAN
  • FastAPI backend with Server-Sent Events progress streaming
  • Click-to-navigate iPhone filesystem and local destinations
  • Real-time transfer progress with file count and byte speed
  • Cancel button, Open in Finder shortcut

Data flow — connect to copy

Step 1
🔌
Plug in iPhone
Lightning or USB-C cable; unlock device and tap "Trust This Computer"
Step 2
🤝
Pair via usbmux
pymobiledevice3 creates a lockdown session over the USB tunnel
Step 3
📂
Browse via AFC
AfcService streams directory listings and file metadata asynchronously
Step 4
☑️
Select files
Pick individual files, folders, or everything — selection shown with size totals
Step 5
⚡
Transfer
Chunked async reads (256 KB) written directly to local disk; progress streamed live

Conflict policies

--conflict skip

Default. If the file already exists at the destination, skip it entirely. Safe for incremental backups.

--conflict overwrite

Replace any existing file with the iPhone version. Use when you want the latest copy unconditionally.

--conflict rename

Keep both files. The new file gets a _1, _2, … suffix appended automatically.
